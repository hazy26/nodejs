Brève historique

Node.js fut créé par Ryan Dahl et fut accessible publiquement avec sa première version en 2009.

C'est un framework open-source, permettant l'exécution de javascript sur de multiples plateformes.

Il gagna rapidement en popularité pour le développement d'applications.

Node.js se base sur la réutilisation de modules, et il est construit sur le moteur javascript V8 de Google.

En 2010, l'arrivé de Node Package Manager (NPM) facilita grandement le partage et la réutilisaion de code entre développeurs.

Aujourd'hui, plus de 3.1 million de paquets sont disponibles sur npm.

Depuis sa première version, Node.js connu plusieurs mises à jour, dont fréquement des améliorations de la performance et de la stabilité du framework

Voici quelques applications notables construitent sur Node.js:

Amazon, Ebay, Linkedin, NASA, Netflix, Paypal, Uber, et plusieurs autres.

Introduction

Comme mentionné plus tôt, Node.js est un framework open-source qui est basé sur la réutilisation de modules.

Chaque développeur peut créer des modules et ensuite les partager aux autres devs grâce à npm (le package manager standard de node.js).

Lorsque Node.js exécute un opération I/O, par exemple accéder à une base de données, au lieu d'attendre la réponse (si elle n'est pas immédiate), Node.js va reprendre l'opération une fois que la réponse revient.

Cela lui permet d'effectuer une multitudes de connexions simultanées dans un même serveur sans devoir gérer la concurrence des threads (qui peut engendrer plusieurs bugs).

Node.js nous permet d'exécuter du javascript tant du côté client que du côté serveur.

Ce qui nous donnes donc la possibilité de faire du full-stack en utilisant seulement que du javascript.

C'est sur le moteur javascript V8 qu'est construit Node.js. Le même moteur qui, lorsque l'on navique sur Chrome, prend notre javascript et l'exécute.

Il est conçu pour être rapide, léger et efficace.

La différence entre Node.js et le navigateur

Les deux utilisent Javascript en tant que leur langage de programmation.

Cependant, construire un application qui fonctionne dans le navigateur est très différente que de construire une application Node.js

L'une des principales différences est que dans le navigateur nous manipulons le DOM, alors que sur Node.js, nous manipulons des modules.

De plus, Node.js nous donne accès au front-end, mais aussi au back-end en utilisant seulement qu'un langage.

La différence entre développement et production

Dans Node.js, il n'y a pas de différence entre un environnement de développement ou bien de production.

Certaines librairies reconnaissent cependant la variable 'NODE_ENV', et initialise cette valeur à 'development'.

Toujours exécuter Node.js avec le 'NODE_EN=production'.

Installation de Node.js

Plusieurs options nous sont offerts.

Voici les plus simples:

Pour les utilisateurs Linux, MacOS et Windows

Le télécharger directement sur:
node.js

Pour les utilisateurs MacOS seulement

1. En utilisant brew

Besoin d'installer brew? Cliquer ici:

Homebrew 
                
                  #télécharger et installer Node.js
                  brew install node@20 (ou la version désirer)

                  node -v  #devrait imprimer la version de node
2. En utilisant nvm (node version manager)

Besoin d'installer nvm cliquer ici:

nvm Github ReadMe 
                
                  # installe nvm (Gestionnaire de version node)
                  curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash

                  # télécharger et installer Node.js (il peut être nécessaire de redémarrer le terminal)
                  nvm install 20

                  node -v #devrait imprimer la version de node

Pour les utilisateurs Windows seulement

1. En utilisant fnm

Besoin d'installer fnm? Cliquer ici:

fnm Github ReadMe 
                
                  #installe fnm (Fast Node Manager)
                  winget install Schniz.fnm
                  
                  #configurer l'environnement fnm
                  fnm env --use-on-cd | Out-String | Invoke-Expression
                  
                  #télécharger et installer Node.js
                  fnm use --install-if-missing 20
                  
                  node -v #devrait imprimer la version de node

Exemple d'utilisation

Voici comment créer un serveur Node.js et afficher un simple message dans le navigateur.

1. Créer un nouveau dossier pour votre projet et un fichier 'exemple.js'
Option 1: manuellement
Option 2: Ouvrez le terminal et entrer les commandes suivantes
                
                  mkdir exemple_nodejs && cd exemple_nodejs

                  touch exemple.js
2. Entrer ensuite dans le terminal la commande 'npm init' (à la racine de votre projet)
                
                  npm init -y

Un fichier 'package.json' devrait s'être créé dans votre projet.

3. Ouvrir le projet dans VScode (ou l'éditeur de votre choix)
4. Copier / Coller le code suivant dans le fichier 'exemple.js'
                
                  const http = require('node:http');

                  const hostname = '127.0.0.1';
                  const port = 3000;
      
                  const server = http.createServer((req, res) => {
                    res.statusCode = 200;
                    res.setHeader('Content-Type', 'text/plain');
                    res.end('Bonjour tout le monde!');
                  });
      
                  server.listen(port, hostname, () => {
                    console.log(`Serveur accessible à http://${hostname}:${port}/`);
                  });
5. Lancer le serveur
Entrer la commande suivante dans le terminal:
                
                  node exemple.js
6. Ouvrer votre navigateur à l'adresse du serveur
